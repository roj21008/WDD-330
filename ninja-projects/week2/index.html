Programming Basics

  • Well-commented code is the hallmark of a ninja programmer
  • It is considered good practice to write each statement on a new line, terminated with a semicolon
  • Words reserved for use by the JavaScript language and cannot be used to name variables
  • JavaScript has a special operator called typeof to find out the type of a value
  • Using “const” is a useful way to deal with long strings or expressions, as it saves us from having to type them over and over again.
  • Why was the new word let introduced in ES6?
    Why not just change the behavior of var?
    Remember that a basic principle of the JavaScript language is that it must remain
    backward compatible. This means that the behavior of var had to remain constant, so it couldn't simply be changed in ES6.
    For that reason, let the new keyword was introduced.
  • If the variable refers to a non-primitive data type, such as an array, function, or object, using const will not make it immutable.
  • Not using let and const makes a global variable.
  • The backslash is used to escape special characters in strings
  • The best approach is to try to be very explicit about the types of values you are working with.
    If you need to change the type, do it manually, rather than relying on type coercion to do it for you

Arrays

  • An array is an ordered list of values
  • Arrays are not primitive values but a special built-in object

Functions

  • A ninja programmer should always declare functions using function literals, function declarations, or function expressions.
  • Don't Repeat Yourself, or DRY, is a programming principle that specifies that each part of a program should only be written once. This avoids duplication and means there is no need to keep multiple pieces of code up to date and in sync.
  • All functions have a read-only property called name.
